self:dd (...)
Dumps the values of variables and tables in the output, then dies.

The dd() stands for "dump and die" and it's a helper function inspired by a PHP framework called Laravel. It's used to dump the values of variables and tables in the output and stop the execution of the script. It's only used for debugging purposes and should never be used in an addon that will be released.

Given that it can't use the Output:out() method, there's no test coverage for dd(). After all it's a test and debugging helper resource.

Parameters:

  • ... The variables and tables to be dumped

Usage:

    dd(someVariable)
dd({ key = 'value' })
dd(someVariable, { key = 'value' })
